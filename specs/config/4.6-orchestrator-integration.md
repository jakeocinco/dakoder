# Task 4.6: Integrate Config into Orchestrator

## Description

The orchestrator reads `.dakoder.toml` from the task's S3 workspace after clone and uses it to configure the workflow.

## Requirements

- Add `dakoder-config` as a dependency of orchestrator
- After workspace creation, load `.dakoder.toml` from S3 (`s3://{bucket}/{taskId}/.dakoder.toml`)
- Parse with `parseConfig()` — if it throws `ConfigError`, fail the task with the error message
- Use `config.workflow.max_iterations` instead of the `MAX_ITERATIONS` env var
- Pass agent config to code-lambda in the invocation payload using the `AgentConfig` type from `dakoder-config`
- If `.dakoder.toml` does not exist in workspace, fail the task with a clear error
- Update `NewTaskEvent` — `repoUrl`, `branch`, `description`, `spec` can now be derived from config where applicable

## Tests

- Mock S3 read of `.dakoder.toml` and verify orchestrator uses config values
- Verify missing config file causes task failure with descriptive error
- Verify invalid config causes task failure with descriptive error
- Verify agent config is passed through to code-lambda invocation payload

## Dependencies

- 4.3, 1.4 (S3 workspace creation)
