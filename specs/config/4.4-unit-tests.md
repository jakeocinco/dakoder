# Task 4.4: Unit Tests

## Description

Comprehensive tests for config parsing and validation.

## Requirements

- Uses `node:test` (matching existing test pattern)
- Test cases:
  - Valid config with all fields → parses correctly, all values match
  - Valid config with only required fields → optional fields get defaults
  - Missing `project.name` → throws `ConfigError` with field name
  - Missing `git.repo_url` → throws `ConfigError` with field name
  - Missing `git.credentials_secret` → throws `ConfigError` with field name
  - Missing `triggers.enabled` → throws `ConfigError` with field name
  - Empty `triggers.enabled` array → throws `ConfigError`
  - Invalid trigger type → throws `ConfigError` with the bad value
  - `max_iterations` is a string → throws `ConfigError` with type info
  - `max_turns` is a string → throws `ConfigError` with type info
  - Invalid TOML syntax → throws `ConfigError`
  - Missing `[agents.code]` section → config has no code agent (uses defaults)
  - Missing `[agents.review]` section → config has no review agent (uses defaults)
- All tests pass with `npm test`

## Dependencies

- 4.1, 4.2, 4.3
