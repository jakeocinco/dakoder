# Task 1.8: Iteration Tracking

## Description

Track and enforce retry count per task.

## Requirements

- Stores iteration count in task metadata (S3 alongside workspace)
- Increments on each retry cycle
- Reads current count before deciding to retry or halt
- Configurable max iteration limit (environment variable or constant)
- Returns current iteration count

## Inputs

- Task ID

## Outputs

- `{ currentIteration: number, maxReached: boolean }`

## Dependencies

- 1.1
- 1.4 (metadata lives in same workspace)
