# Task 1.7: Callback Handler

## Description

Process async results returned from worker Lambdas and decide next action.

## Requirements

- Parses callback payload from Review Lambda or Build Lambda (on failure)
- Determines outcome: approved, rejected, or build failed
- On approved: triggers finalization
- On rejected/failed: increments iteration, rebuilds prompt, invokes Code Lambda
- On max retries exceeded: halts task and reports failure
- Validates callback payload structure

## Inputs

- Callback payload: `{ taskId, status, feedback?, buildLogs?, changedFiles? }`

## Outputs

- Next action taken (invoke Code Lambda, finalize, or halt)
