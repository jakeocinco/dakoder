# Task 0.3: Lambda Functions

## Description

Define the four Lambda functions (orchestrator, code, build, review).

## Requirements

- Each function named `{stackTag}-dakoder-{functionName}` (e.g., `dakoder-dakoder-orchestrator`)
- Node.js runtime (TypeScript compiled)
- Each function granted read/write access to the S3 bucket
- Environment variables set from `.dakoder.toml` config (read at synth time):
  - `BUCKET_NAME` — S3 bucket name
  - `STACK_TAG` — project name from config
  - `REPO_URL` — from `git.repo_url`
  - `DEFAULT_BRANCH` — from `git.default_branch`
  - `BRANCH_PREFIX` — from `git.branch_prefix`
  - `GIT_CREDENTIALS_SECRET` — from `git.credentials_secret`
  - `MAX_ITERATIONS` — from `workflow.max_iterations`
  - Cross-references: `CODE_FUNCTION_NAME`, `BUILD_FUNCTION_NAME`, `REVIEW_FUNCTION_NAME`, `ORCHESTRATOR_FUNCTION_NAME`
- Each function has permission to invoke the other Lambdas
- Orchestrator Lambda granted `secretsmanager:GetSecretValue` on the `credentials_secret` ARN

## Inputs

- Stack tag (from config `project.name`)
- S3 bucket reference
- Parsed `DakoderConfig`

## Outputs

- Four Lambda function resources

## Dependencies

- 0.1
- 0.2
