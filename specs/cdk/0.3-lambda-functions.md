# Task 0.3: Lambda Functions

## Description

Define the four Lambda functions (orchestrator, code, build, review).

## Requirements

- Each function named `{stackTag}-{functionName}` (e.g., `dakoder-orchestrator`)
- Node.js runtime (TypeScript compiled)
- Each function granted read/write access to the S3 bucket
- Environment variables include: bucket name, stack tag, other function names (for async invocation)
- Each function has permission to invoke the other Lambdas
- Placeholder handler code (actual implementation comes from orchestrator/worker tasks)

## Inputs

- Stack tag
- S3 bucket reference

## Outputs

- Four Lambda function resources

- 0.2
