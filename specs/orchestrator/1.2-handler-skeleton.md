# Task 1.2: Handler Skeleton

## Description

Lambda handler entry point that parses incoming events and routes to the appropriate logic.

## Requirements

- Exports a Lambda handler function matching AWS Lambda signature
- Distinguishes between a new task trigger and a worker callback
- New task event: `{ type: "new-task", description, spec }`
  - `repoUrl` and `branch` come from env vars (`REPO_URL`, `DEFAULT_BRANCH`), not the event
- Callback event: `{ type: "callback", taskId, status, feedback?, buildLogs?, description, spec }`
- Routes new tasks to workspace creation flow
- Routes callbacks to the callback handler
- Returns appropriate responses for each path
- Rejects malformed events with a clear error

## Inputs

- Lambda event (trigger event or worker callback payload)
- Lambda context

## Outputs

- Routed execution to correct internal logic
- Error response for invalid events
