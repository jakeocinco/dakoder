# Task 1.10: Error Handling

## Description

Handle failure cases gracefully across all orchestrator logic.

## Requirements

- Catches and logs Lambda invoke timeouts
- Rejects malformed payloads with clear error messages
- Handles S3 access failures (permissions, missing bucket)
- Handles git clone failures (bad URL, auth issues, network)
- Logs errors with task ID and context for debugging
- Marks task as failed with reason in metadata
- Does not crash the Lambda on unexpected errors

## Inputs

- Any error thrown during orchestrator execution

## Outputs

- Error logged with context
- Task metadata updated with failure reason

## Dependencies

- 1.1
- 1.4 (writes failure to metadata)
