# Task 4.3: Parse and Validate

## Description

Implement the `parseConfig(raw: string): DakoderConfig` function that parses TOML and validates all fields.

## Requirements

- Parses raw TOML string using `smol-toml`
- Validates all required fields are present:
  - `project.name`
  - `git.repo_url`
  - `git.credentials_secret`
  - `triggers.enabled`
- Validates types:
  - String fields are strings
  - Number fields (`max_iterations`, `max_turns`) are numbers
  - `triggers.enabled` is a non-empty array of strings
- Validates trigger values are in the allowed set (`"pr_comment"`, `"merge"`)
- Applies defaults for optional fields
- `[agents.code]` and `[agents.review]` sections are fully optional — omitting them means use Claude defaults
- Throws `ConfigError` with descriptive message on any failure:
  - `"Missing required field: project.name"`
  - `"Invalid type for workflow.max_iterations: expected number, got string"`
  - `"Invalid trigger type: 'push'. Valid types: pr_comment, merge"`
  - `"triggers.enabled must not be empty"`
- Returns fully resolved `DakoderConfig` object

## Dependencies

- 4.1, 4.2
