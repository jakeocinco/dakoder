# Task 4.7: Integrate Config into Code Lambda

## Description

Code lambda reads agent config from the event payload and applies it when invoking the Claude agent.

## Requirements

- Update `CodeLambdaEvent` to include an optional `agentConfig` field typed as `AgentConfig` (imported from `dakoder-config`)
- If `agentConfig.promptFile` is specified, read that file from the local workspace and prepend its content to the task prompt
- Use `agentConfig.maxTurns` when calling `query()` (fall back to `25` if not provided)
- If `promptFile` is specified but the file doesn't exist in workspace, log a warning and continue without it (don't fail)

## Tests

- Custom `promptFile` content is loaded and prepended to prompt
- `maxTurns` from config is passed to `query()` options
- Defaults work when no `agentConfig` is passed in event
- Missing prompt file logs warning but doesn't fail the invocation

## Dependencies

- 4.2 (types), 3.4 (agent invocation)
